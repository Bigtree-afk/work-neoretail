⑥ 업로드 흐름 — 이미지 / 파일 자동 분기

이미지 흐름

파일 선택 → type 검사 (image/*) → HEIC→JPG → 리사이즈 1600px → JPEG q=0.85 → /api/upload → R2 PUT → job.attachments[] 추가

파일 흐름 (이미지 아닌 경우)

파일 선택 → 확장자 화이트리스트 → 크기 검사 (≤50MB) → /api/upload (원본 그대로) → R2 PUT → job.attachments[] 추가

단계별 처리

단계 위치 대상 처리
파일 선택클라이언트전체input change → File[] 추출
분기 검사클라이언트전체file.type 으로 image / file 분기
HEIC 변환클라이언트이미지type=='image/heic' → heic2any() → image/jpeg
리사이즈+압축클라이언트이미지canvas max 1600px → toBlob('image/jpeg', 0.85)
화이트리스트클라이언트파일pdf/hwp/hwpx/docx/xlsx/pptx/zip 만 허용 (exe/bat 등 차단)
크기 검사클라이언트파일≤50MB (LINE 호환 한도)
업로드네트워크전체FormData POST → /api/upload (XHR progress)
R2 PUT서버이미지키: ns-images/{yyyy}/{MM}/{uuid}.jpg
R2 PUT서버파일키: ns-files/{yyyy}/{MM}/{uuid}.{ext} · 원본 파일명 메타 보존
job 기록클라이언트전체{kind:'image'|'file', key, url, name, size, ...} → attachments[]
R2 prefix 분리: ns-images/... vs ns-files/... — 휴지통/통계도 분리 관리.
이미지는 항상 JPG 변환 (PNG/WebP/GIF 도) — LINE Image Message 호환 + 용량 절감.
