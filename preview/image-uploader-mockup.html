④ 업로드 흐름 (1장 기준)

파일 선택 → HEIC? → 리사이즈 (≤1600px) → JPEG 압축 (q=0.85) → /api/upload-image POST → R2 PUT → job.images[] 추가

단계별 처리

단계 위치 처리
파일 선택클라이언트input change 이벤트 → File[] 추출
HEIC 변환클라이언트file.type==='image/heic' → heic2any() → Blob (image/jpeg)
리사이즈클라이언트canvas: max(width,height) > 1600 이면 비율 유지 축소
압축클라이언트canvas.toBlob('image/jpeg', 0.85) → 200~400KB
업로드네트워크FormData POST → /api/upload-image (XHR 로 progress 추적)
R2 PUT서버 (Worker)키 = ns-images/{yyyy}/{MM}/{uuid}.jpg → R2 bucket
job 기록클라이언트{key,url,w,h,size,uploadedAt,by} → thread item.images[]
HEIC 외에도 PNG/WebP/GIF 도 동일하게 JPEG 로 변환 → R2 에는 항상 jpg 만 저장 (LINE 호환).
원본 보존이 필요하면 추후 옵션 추가 가능.

⑦ API 명세 — /api/upload-image

요청

POST /api/upload-image
Content-Type: multipart/form-data
Cookie: ns_session=...  (로그인 검증)

FormData {
  file: Blob (image/jpeg, ≤ 10MB)
  jobId: 'JOB-MP9X12'        (선택)
  category: 'as'             (재고/as/신규/van/소모품/메모)
  threadId: 'th-xxx'         (선택)
}

응답 (성공)

200 OK
{
  "ok": true,
  "key": "ns-images/2026/05/abc123def456.jpg",
  "url": "https://pub-xxxx.r2.dev/ns-images/2026/05/abc123def456.jpg",
  "size": 245380,
  "w": 1600,
  "h": 1200,
  "uploadedAt": "2026-05-25T13:01:23+09:00",
  "uploadedBy": "kimgisa"
}

응답 (실패)

400 Bad Request   { ok:false, error:"unsupported_type" }
401 Unauthorized  { ok:false, error:"login_required" }
413 Payload Too Large  { ok:false, error:"too_large", maxMB:10 }
500 Internal Error  { ok:false, error:"r2_put_failed", detail:"..." }

삭제 — /api/delete-image

POST /api/delete-image
{ "key": "ns-images/2026/05/abc123def456.jpg" }

→ 실제 R2 삭제 X. 휴지통 prefix 로 이동:
  ns-images/_trash/2026/05/abc123def456.jpg
  (30일 후 cron 으로 영구 삭제)
삭제는 휴지통 30일 — 실수 복구 가능. 관리자 페이지에서 휴지통 조회/복원/즉시삭제 UI 추가 예정.
