⑩ 구현 단계 (재고조사 먼저 → 5종 확장)

Phase 1 — 인프라 (1회)

  1. /api/line-send endpoint (Cloudflare Pages Function)
    • POST { roomId, text, jobId, category, status }
    • LINE Messaging API POST /v2/bot/message/push 호출
    • 발송 직전 sha256(roomId+':'+text).slice(0,16) 생성
    • KV line_outbound_sig:<hash> 에 { jobId, sentAt, by, category, status } 저장 (TTL 10분)
    • 응답: { ok, sig, sentAt }
  2. line-webhook.js 수신 시 — 메시지 본문 해시 매칭 → KV hit 시 processedStatus='self_echo'
  3. line-parse-cron.js 에서 self_echo skip
  4. 관리자 페이지에 LINE 그룹 매핑 표 + 테스트 발송 UI (위 표)

Phase 2 — 재고조사 적용 (먼저)

  1. 등록 시 (_stocktakeSaveEditor 신규) → 요청접수 메시지 발송 옵션
  2. 진행 추가 — 새 [+ 진행 기록] 버튼 + 공통 패널 (text + LINE 토글)
  3. 완료 처리 (_stocktakeMarkDone) → 완료 메시지 옵션
  4. rec.lineHistory[] 에 발송 이력 누적 + 카드에 📡 N회 칩

Phase 3 — 확장 (AS / 신규 / VAN / 소모품)

  1. 공통 컴포넌트 추출: _renderLineSendPanel(jobId, category, status, suggestedText)
  2. 각 카테고리 thread 렌더러의 [+ 진행] [✅ 완료] 버튼이 위 함수 호출
  3. 발송 결과 → job.lineHistory[] 에 누적, hub 카드 칩 갱신
  4. 매장 상세 / 작업 편집 모달 우상단 📡 N회 발송 클릭 시 이력 모달

Phase 4 — 안전망 / 운영

  1. 발송 실패 retry (3회) + 실패 시 빨간 배지
  2. 발송 권한 — 본인 또는 admin 만 발송 가능 (실수 방지)
  3. 매장/거래처별 LINE 그룹 오버라이드 (선택)
  4. 발송 메시지 템플릿 관리자 화면 (카테고리별 포맷 편집)
