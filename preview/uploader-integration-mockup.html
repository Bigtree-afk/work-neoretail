작성 영역 (위 ①~⑤)

  • 전체 업로더 (64px 카드 + ＋ 추가 버튼) — 편집 가능
  • NS_UPLOAD.mount(div, { readonly:false, onChange })

표시 영역 (thread item / 카드 리스트)

  • 작은 썸네일 strip (36px) — 클릭 시 라이트박스
  • 너무 많으면 5개 + +N 표시 (전체는 라이트박스에서)
  • 구현: _renderAttStrip(attachments, limit=5) 별도 헬퍼

🚀 통합 작업 순서 (Phase 2)

  1. 재고조사 등록 폼 — _stocktakeOpenEditor() 안에 uploader mount
  2. 재고조사 진행/완료 패널 — 진행 추가 폼 안에 mount, 저장 시 thread item.attachments[]
  3. 썸네일 strip 헬퍼 — _renderAttStrip(attachments, opts) (목록 카드 + thread item 공용)
  4. AS 편집 모달 — ROOT/child 작성 영역 각각에 mount
  5. 신규 등록/편집 폼 — 요청 내용 textarea 아래
  6. VAN 업무 모달 — 업무 내용 textarea 아래
  7. 저장 흐름 — onChange 콜백으로 ns_jobs / cloud KV 에 attachments 보존
  8. (선택) 매장 상세 — 첨부 갤러리 탭
각 단계마다 readonly 모드 (기존 데이터 표시) 와 편집 모드 (새 첨부 추가) 가 자연스럽게 전환되어야 함.
