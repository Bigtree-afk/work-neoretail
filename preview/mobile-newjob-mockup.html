⚡ 빠른 발송  ①홈 → FAB📡 → ⑥빠른발송 (단계 선택 → 사진 → 한 줄)

📐 디자인 원칙

  • 일정 트리플 시각화 — 설치→가오픈→오픈 3단계 타임라인 항상 노출 (D-카운트로 임박 강조)
  • 장비 카탈로그 빠른 진입 — 카테고리 칩 (POS/VAN/키오스크/주변기기) + 검색 + +/− 수량 조절
  • VAN TID 입력 최적화 — inputmode="numeric" + 큰 폰트(14.5px) + letter-spacing 으로 8자리 가독성
  • VAN 카드 분리 — KOCES/NICE/KIS/KSNET/간편/카카오 각각 카드 — 펼침/접힘 + 진행률 표시
  • 3-스텝 풀폼 — 매장/일정 → 장비 → VAN. 각 스텝마다 임시저장 가능
  • FAB 2개 디자인 — 일반 ＋ (신규 등록) + LINE 📡 (빠른 사진+한줄 발송)
  • LINE 발송 디폴트 ON — 단계별 자동 발송 (해제 가능). 단계 태그 (설치/가오픈/오픈/이슈) 자동 삽입
  • 일정 임박 시각 강조 — D-2/D-1 등 빨간 배지 + 오픈 임박 탭 우선 노출
  • 큰 터치 영역 + 30자 카운터 — 야외/현장 입력 가독성

🔧 기술 노트

  • 같은 index.html 사용 — 별도 모바일 앱 아님. CSS 미디어쿼리로 모바일 모드 분기
  • @media (max-width: 640px) 에서 활성화: 큰 버튼, 바텀시트, FAB, 풀폼 스텝퍼
  • 일정 트리플: .timeline-trio 컴포넌트 → 설치/가오픈/오픈 3-state (done/cur/future)
  • VAN 입력: jobVandocs 컬렉션 — {van:'KOCES', tid:'22041897', serial:'KS7K-...', terminal_no:1}
  • 장비 카탈로그: equipment_catalog + job_equipment 조인 (qty 포함)
  • 카메라 인풋: <input type="file" accept="image/*" capture="environment">
  • D-카운트 자동 계산: 오픈일 - 오늘 ≤ 2일 이면 빨간 배지 + 오픈 임박 탭 표시
  • VAN 진행률: (입력완료 단말수 / 전체 단말수) × 100 카드별 + 전체
  • 오프라인 대비: TID/Serial 입력은 IndexedDB 임시 저장 → 온라인 복귀 시 동기화
  • LINE 메시지 단계 태그: [설치] / [가오픈] / [오픈] / [이슈] 자동 prefix
