🗂️ 화면 흐름 (사용자 시나리오)

📥 신규 등록  ①홈 → ⑥매장선택 → ⑨등록① → ⑩등록② → ⑪정산입력 → 저장+📡
🚗 현장 진행  ①홈 → 카드탭 → ⑫thread → ②진행추가시트 → 📡 발송
⚡ 빠른 발송  ①홈 → FAB📡 → ③빠른발송 → 📡 (등록 별도)
✅ 완료/정산  ⑫thread → 완료버튼 → ④완료시트 → 📡 발송
📎 첨부  ⑫thread 또는 ⑩등록② → ⑬첨부관리 (다중 선택)
📜 이력  ⑫thread → 📡뱃지 탭 → ⑤LINE이력

📐 디자인 원칙

  • 한 손 사용 — 주요 액션 (저장/발송/촬영) 은 항상 화면 하단 영역에 배치
  • 큰 터치 영역 — 모든 버튼 최소 44px 높이 (Apple HIG 기준)
  • 카메라 즉시 진입 — [📸 촬영] 버튼은 OS 카메라 다이얼로그 통과 없이 바로 띄움
  • FAB 2개 디자인 — 일반 ＋ (작업 등록) + LINE 📡 (빠른 사진+한줄 발송)
  • 스텝퍼 — 요청 → 진행 → 완료 시각 표시. 완료된 단계는 ✓ 로 회색 → 녹색
  • LINE 발송 디폴트 ON — 현장 작업자는 매번 토글하지 않아도 자동 발송 (해제 가능)
  • 미리보기 항상 노출 — LINE 으로 어떤 메시지가 갈지 발송 전 확인
  • 30자 카운터 — 메시지 길이 실시간 표시 (≤30 권장)
  • 30px 이상 폰트 + 13px 본문 — 야외에서도 보이도록

🔧 기술 노트

  • 같은 index.html 사용 — 별도 모바일 앱 아님. CSS 미디어쿼리로 모바일 모드 분기
  • @media (max-width: 640px) 에서 활성화: 큰 버튼, 바텀시트, FAB
  • 카메라 인풋: <input type="file" accept="image/*" capture="environment">
  • iOS Safari Add to Home Screen 으로 앱처럼 사용 (PWA manifest 추가 가능)
  • 오프라인 대비: 사진은 IndexedDB 임시 저장 → 온라인 복귀 시 R2 업로드
  • 위치 정보 (선택): GPS 로 매장 자동 매칭 (배포 후 옵션)
