데이터 모델 & 구현 흐름
1. thread 구조 (이미 적용됨)
job.thread = [ {요청접수 ROOT}, {진행 child}*N, {완료 child} ]
ROOT (parentId:null) = 요청접수 / children (parentId:rootId) = 진행 또는 완료

2. 카테고리별 동일 UI
재고조사 / AS / 신규 / VAN / 소모품 — 모두 같은 thread 구조 + 동일 UI 컴포넌트
카테고리 칩만 다름, 흐름 동일

3. LINE 발송 — 각 entry 추가 시점
· 요청접수 등록 → 카테고리에 매핑된 LINE 그룹으로 발송
· 진행 추가 → 같은 LINE 그룹에 진행 메시지
· 완료 처리 → 완료 메시지
· 각 단계 발송 여부는 사용자 선택 (체크박스, 기본 ON)

4. Self-echo 차단
① 발송 직전 — sha256(roomId + ':' + text).slice(0,16) 해시 → KV (TTL 10분) 저장
② 봇 본인 메시지 webhook 으로 회수 — KV 매칭 → processedStatus='self_echo' → 등록 대기 미노출
③ 발송 이력은 job.lineHistory[] = [{at, status, text, roomId, sig}] 에 남음

5. 최소 구현 (재고조사 먼저)
① /api/line-send endpoint — POST {roomId, text} → LINE Push API + 해시 KV
② line-webhook.js 에서 해시 매칭 시 self_echo
③ 재고조사 등록/진행/완료 3 트리거에 LINE 발송 옵션 추가
④ 발송 이력 UI — 매장 상세 모달 또는 재고조사 카드
