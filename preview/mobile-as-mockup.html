📐 디자인 원칙

  • 긴급 케이스 빨강 강조 — 처리 예정일 3시간 이내 = 카드 좌측 빨강 바 + 🚨 펄스 배지 + D-day 빨강
  • D-day 색상 그라데이션 — D-3+ 녹색 → D-1 노랑 → D-0/3h 이내 빨강
  • 한 손 사용 — 주요 액션(출동/완료/발송)은 항상 화면 하단 영역에 배치
  • 큰 터치 영역 — 모든 버튼 최소 44px 높이 (Apple HIG 기준)
  • 카메라 즉시 진입 — [📸 촬영] 버튼은 OS 카메라 다이얼로그 통과 없이 바로 띄움 (capture="environment")
  • FAB 2개 — 일반 ＋ (AS 등록) + LINE 📡 (빠른 사진+한줄 발송)
  • 스텝퍼 — 요청 → 진행 → 완료. 회색 → 파랑(현재) → 녹색(완료)
  • 📞 전화 버튼 직결 — 긴급 케이스에선 매장 담당자 번호 1탭 호출 (tel:)
  • LINE 발송 디폴트 ON — 출동/진행/완료 매 단계 자동 발송 (해제 가능), 미리보기 항상 노출
  • 대상 장비 칩 — VAN/POS/키오스크/기타 다중선택, 카드에 작은 회색 칩으로 노출
  • 투입 장비 카탈로그 — 자주 쓰는 부품은 카탈로그에서 ✅ 토글 + 수량 입력, 합계 자동 산출

🔧 기술 노트

  • 같은 index.html 사용 — 별도 모바일 앱 아님. CSS 미디어쿼리로 모바일 모드 분기
  • @media (max-width: 640px) 에서 활성화: 큰 버튼, 바텀시트, FAB
  • 카메라 인풋: <input type="file" accept="image/*" capture="environment">
  • 전화 호출: <a href="tel:01012345678"> — 긴급 카드 📞 버튼에서 1탭 진입
  • D-day 계산: 처리예정일과 현재 시각의 ms 차이로 색상 클래스 분기 (safe/near/urgent)
  • 긴급 임계: 3시간 이내 = urgent, 24시간 이내 = near, 그 외 = safe
  • 투입 장비 합계: items.reduce((s,i) => s + i.qty * i.price, 0) — 카탈로그 단가는 config 에 저장
  • LINE 메시지 포맷: [AS] {매장명} : {일시} ; {내용} · 🧰 {장비요약} · 📷 {N장}
  • iOS Safari Add to Home Screen 으로 앱처럼 사용 (PWA manifest 추가 가능)
  • 오프라인 대비: 사진은 IndexedDB 임시 저장 → 온라인 복귀 시 R2 업로드
  • 긴급 케이스 푸시 알림 (선택): Web Push API + service worker → D-0/3h 이내 자동 알림
