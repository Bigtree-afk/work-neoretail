🗂️ 화면 흐름 (사용자 시나리오)

📥 VAN 신규 등록  ①홈 → FAB＋ → ⑧매장선택 (prefill) → ②풀폼① → ③풀폼② → ④풀폼③ → 저장+📡
🚗 가맹 진행 추가  ①홈 → 카드탭 → ⑤thread → 진행추가 시트 → 📡 발송
⚡ 빠른 발송  ①홈 → FAB📡 → ⑦빠른발송 (매장+VAN+사진+한줄) → 📡
📊 가맹 상태 확인  ①홈 → 카드탭 → ⑥VAN진행상태 → TID 복사 / 수정
🔁 재신고/변경  ⑧매장선택 (prefill ✨) → 기존 KOCES TID 자동 입력 → ③풀폼② 수정
✅ 완료/정산  ⑤thread → 완료버튼 → ⑥VAN상태 (100%) → 📡 발송

📐 디자인 원칙

  • TID/Serial 입력 단순화 — 복붙 버튼 📋 항상 노출, 카메라 OCR 📸 보조 (단말 라벨 자동 인식)
  • 매장 선택 시 prefill ✨ — 기존 VAN 프로필 있으면 KOCES/NICE TID 등 자동 채움 (재신고/변경 시 핵심)
  • 가맹 신청일 = 오늘 (자동) — 신청일 필드 디폴트로 오늘 날짜 입력
  • 가맹 진행률 가시화 — 카드/홈/상세 모두 진행률 바로 표시 (영업일 기반 추정)
  • VAN 4종 복수 체크 — KOCES/NICE/KIS/KSNET 각각 독립 토글 + TID/Serial 행 펼침
  • 카테고리 칩 — 신규등록/재신고/정산/계약/변경 5개 가로 스크롤 칩 (홈 필터)
  • LINE 발송 디폴트 ON — 매장+카테고리+VAN사+한줄 자동 합성 미리보기
  • 한 손 사용 — 주요 액션은 항상 화면 하단, 44px+ 터치 영역
  • VAN 보라 컬러 시그니처 — #7C3AED 로 재고(파랑)·AS와 시각적 구분

🔧 기술 노트

  • 매장 prefill 데이터 — store.vanProfile = {kocesTid, kocesSerial, niceTid, ...} 매장 모델에 캐시
  • TID OCR — <input type="file" accept="image/*" capture="environment"> + Tesseract.js / Vision API 로 숫자 추출
  • 복사 버튼 — navigator.clipboard.writeText(tid) + 토스트 알림
  • 가맹 진행률 계산 — (today - 신청일) / 영업일 5, 단 완료일 입력 시 100%
  • 카테고리 칩 — URL query ?cat=new|re|settle|contract|change 필터 연동
  • LINE 발송 페이로드 — {store, category, van, msg, photos[]} → /api/line-send
  • 오프라인 대비 — 단말 사진은 IndexedDB 임시 저장 → 온라인 복귀 시 R2 업로드
  • 같은 index.html — 별도 앱 아님. @media (max-width:640px) 분기로 모바일 활성화
