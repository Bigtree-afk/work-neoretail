📤 Phase 1 — 업로더 스모크 테스트

R2 버킷 ns-attachments 가 설정된 후 /api/upload 가 응답하면 실제 동작 확인 가능.
PC: 파일 선택 다이얼로그 / 모바일: OS 시트 (촬영/갤러리/파일) 자동 표시.

1. 업로더 마운트

2. 현재 attachments[] 상태

(아직 비어있음)

3. 수동 라이트박스 테스트 (외부 URL)

4. R2 설정 안내

① Cloudflare Dashboard → R2 → Create bucket: ns-attachments
② 버킷 → Settings → Public access → r2.dev subdomain 활성화
   → URL 복사 (예: https://pub-xxxxx.r2.dev)
③ Pages 프로젝트 → Settings → Environment variables:
   PUBLIC_R2_BASE = https://pub-xxxxx.r2.dev
④ Pages 프로젝트 → Settings → Functions → R2 bucket bindings:
   Variable name: ATTACHMENTS, Bucket: ns-attachments
⑤ 배포 → /api/upload 호출 테스트
PUBLIC_R2_BASE 미설정 시 자동으로 /api/file?key=... 프록시로 폴백.
