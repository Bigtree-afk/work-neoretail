📥 신규 발송  ①홈 → FAB＋ → ②등록① (매장 + 품목 ⭐) → ⑧카탈로그 → ③등록② (송장 OCR + 첨부) → ④수금 → 저장+📡

⚠️ 미수금 처리  ①홈 (미수금 칩) 또는 통계카드 → ⑥미수금 내역 → 수금/독촉/전화 → ④수금 처리
↻ 재주문 (단골)  ⑧카탈로그 → 매장별 최근주문 → ↻재주문 → ③등록② → ④수금
📡 LINE  단계마다 디폴트 ON · 미리보기 항상 노출 · 토글로 해제 가능

📐 디자인 원칙

  • 자주 쓰는 품목 ⭐즐겨찾기 — 필드 직원이 한 손으로 3탭 안에 발송 등록 (홈 → ＋ → 매장 + 즐겨찾기 3개 선택 → 저장)
  • 미수금 색상 알림 — 30일 🟡 / 60일 🟠 / 90일 🔴 단계별 경고. 홈 통계 카드와 ⑥내역 모두 일관 색상
  • 송장번호 OCR — [📸 OCR] 버튼 한 번에 카메라 → 텍스트 인식 → 신뢰도 표시 (수기 검토 가능)
  • 카메라 즉시 진입 — [📸 촬영] OS 카메라 다이얼로그 통과 없이 바로 띄움 (영수증/송장/배송 모두 공통)
  • LINE 발송 디폴트 ON — 매번 토글하지 않아도 자동 발송 (해제는 가능)
  • FAB 2개 — ＋ (신규 발송 등록) / 📡 (빠른 LINE 발송)
  • 스텝퍼 3단 — 매장/품목 → 배송 → 수금 (3/3). 완료된 단계는 ✓ 녹색
  • 한 손 사용 — 주요 액션 (저장/발송/촬영) 항상 화면 하단 영역
  • 금액 자동 계산 — 수량 × 단가 → 즉시 합계 갱신 (Excel처럼)
  • 미리보기 항상 노출 — LINE 으로 어떤 메시지가 갈지 발송 전 확인

🔧 기술 노트

  • 재고조사 모듈과 같은 디자인 시스템 — device/screen/appbar/card/stepper/fab/sheet 동일 컴포넌트 재사용
  • 모바일 분기: @media (max-width: 640px) 에서 활성화 (FAB + 바텀시트 + 큰 버튼)
  • OCR: 클라이언트 측 Tesseract.js 또는 Cloudflare Worker AI @cf/microsoft/resnet-50 기반. 송장번호는 정규식 (택배사별 패턴) 검증
  • 카메라 인풋: <input type="file" accept="image/*" capture="environment">
  • 즐겨찾기: 각 사용자별 최근 주문 빈도 기준 자동 정렬 + 수동 ★ 토글
  • 미수금 경과일수: 발송일/예정일 기준 (now - issued_at) / 86400 → CSS 클래스 a30/a60/a90 매핑
  • 오프라인 대비: 사진/입력은 IndexedDB 임시 저장 → 온라인 복귀 시 R2 + DB 일괄 업로드
  • 택배사 송장 API: 도착 확인은 트래킹 API (CJ/한진/우체국) 폴링 또는 webhook
  • iOS Safari PWA: Add to Home Screen 으로 앱처럼 사용 (manifest 추가 가능)
